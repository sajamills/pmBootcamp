## Codebase Patterns
- All main pages live in `app/(main)/` route group; `app/login/` is standalone (no sidebar)
- `useProgress()` from `contexts/ProgressContext.tsx` provides `{ progress, isOwner, setLink, toggle, refreshProgress }`
- `LinkEditor` is defined inline in `app/(main)/portfolio/page.tsx` â€” copy it inline into admin page (no shared file)
- Week page (`app/(main)/week/[week]/page.tsx`) is a server component â€” no hooks available
- Share page (`app/(main)/share/page.tsx`) is a server component that calls `getProgressFromKV()` directly
- Portfolio page is a client component (`"use client"`); `deliverables` splits into `completed` and `upcoming` arrays
- Sidebar nav lives in `components/Sidebar.tsx`; isOwner conditional on line 67 for progress label
- `progress.completedTasks` stores task completion as `{ [taskId]: ISODateString }` values
- `process.env.NEXT_PUBLIC_RESUME_URL` â€” not yet set; resume button should render only when this is truthy
- `curriculum` imported from `@/data/curriculum`; each week has `week.week`, `week.theme`, `week.goal`, `week.portfolioDeliverable`
