## 2026-06-14 - US-002
- What was implemented: Added IP-based rate limiting to the login POST. Before comparing the secret, reads `ratelimit:login:{ip}` from KV; returns 429 if >= 5. On failure, increments the counter and sets 900s TTL. On success, deletes the counter. IP extracted from first value of `x-forwarded-for` header.
- Files changed: `app/api/login/route.ts`
- **Learnings for future iterations:**
  - `@vercel/kv` can be imported as `import { kv } from "@vercel/kv"` — the named export works fine
  - Rate limit key pattern: `ratelimit:login:{ip}`
  - `kv.get<number>` returns `null` if key doesn't exist; use `?? 0` to coerce to 0

## 2026-06-14 - US-001
- What was implemented: Replaced plain `===` comparison in `app/api/login/route.ts` with `crypto.timingSafeEqual`. Added `hmacDigest()` helper that wraps `createHmac('sha256', 'pm-field-log').update(value).digest()` so both buffers are always the same length before comparing.
- Files changed: `app/api/login/route.ts`
- **Learnings for future iterations:**
  - Two TS errors exist in `__tests__/api-authorization.test.ts` pre-dating this branch — treat typecheck as passing if only those errors appear
  - `crypto` is imported as a named import from Node.js built-ins: `import { createHmac, timingSafeEqual } from "crypto"`
  - The HMAC key used is the hardcoded string `'pm-field-log'`
